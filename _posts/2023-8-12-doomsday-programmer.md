Programming is a noble and useful craft that can produce wonderful things, such as machines, games, or art. But many programmers have fallen into the trap of using too many crutches, such as libraries, frameworks, or editors, that make their work easier but also weaker, heavier, or more dependent. In this essay, I will argue for the doomsday programming ideology, which urges programmers to use no crutches and only the bare essentials to make their work. I will show the reasons, advantages, and difficulties of this ideology, and give some examples of doomsday programming in action.



The main reason for the doomsday programming ideology is to be ready for the worst possible situation. Imagine a world where the internet is gone, the computers are broken, and the software is corrupted. How would you survive as a programmer? How would you fix the problems? How would you create new things? If you have been using crutches all along, you would be helpless and hopeless. You would not know how to write code without copying from others, how to run code without installing dependencies, or how to debug code without using tools. But if you have been using no crutches and only the essentials, you would be able to cope and adapt. You would know how to write code from scratch, how to run code with minimal resources, or how to debug code with logic and intuition.

The main advantage of the doomsday programming ideology is to make your work more robust, lean, and independent. By using no crutches and only the essentials, you avoid many problems that plague modern programming. You avoid bugs that come from faulty or outdated dependencies. You avoid bloat that comes from unnecessary or redundant features. You avoid dependence that comes from relying on external or proprietary services. Instead, you create code that is more reliable, efficient, and self-contained. You create code that works on any platform, any device, or any situation. You create code that you own and control.



The main difficulty of the doomsday programming ideology is to overcome the temptation and pressure of using crutches. It is not easy to resist the allure of using libraries that promise to solve your problems in one line of code, frameworks that promise to make your work faster and easier, or editors that promise to make your code more beautiful and error-free. It is not easy to ignore the criticism of others who may mock your work as primitive, outdated, or unprofessional. It is not easy to endure the hardship of writing code without help, running code without comfort, or debugging code without guidance. But it is worth it. It is worth it because you learn more, you improve more, and you achieve more.

Doomsday programming is an ideology that advocates for using no crutches and only the essentials to make your work. Doomsday programming prepares you for the worst-case scenario, makes your work more robust, lean, and independent, and challenges you to overcome the temptation and pressure of using crutches. Doomsday programming is a defence of simplicity, a celebration of skill, and a pursuit of excellence.
